RecipientService

public with sharing class RecipientService
DescriptionThis class is the entry point for DocuSign for Salesforce envelope recipient management. It provides methods to retrieve Salesforce source recipients and add or remove recipients from an envelope.

  searchRecipients

public static List<Recipient> searchRecipients(final SObjectType type, final String firstName, final String lastName)
DescriptionSearches for DocuSign envelope recipients from Salesforce source objects.

By default, the Salesforce Contact, Lead, and User types are supported. More types may be supported via plugins.

Search behavior is defined and stored in account settings. See Recipient.SearchBehavior and AccountService.Settings for more details.

The caller must be assigned the DocuSign User permission set.

ReturnsA list of envelope recipients.
Parameters

lastNameThe last name to match. Ignored for compound search behaviors.

  validate

public static ValidationResult validate(final List<Recipient> recipients, final Boolean sendNow)
DescriptionValidates a list of DocuSign envelope recipients. They are checked for the following:
  • At least one recipient is defined.
  • If sending now and the recipient is not a signing group, the email address must not be blank.
  • If not sending now and the email address is blank, the role name must not be blank.

This method is intended for DocuSign internal use only.

ReturnsThe result of the validation.
Parameters
recipientsThe recipients to validate.
sendNowWhether the intention is to send the envelope now or to continue preparation.
